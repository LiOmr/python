### Теоретическая часть

### Кластерный анализ
Кластеризация (англ. cluster analysis) — это задача группировки множества объектов на подмножества (кластеры) таким образом, чтобы объекты из одного кластера были более похожи друг на друга, чем на объекты из других кластеров по какому-либо критерию. 

Задача кластеризации часто применяется для первичного анализа статистических данных с целью выявления групп, объединенных общими критериями. После этого можно проводить дальнейшие исследования по формализации этих групп и введению классов, что приводит к задаче классификации.

### Методы кластеризации
Среди самых распространенных методов кластеризации выделяются:

1. KMeans (k-средних)  
   Это самый простой алгоритм кластеризации. Его основная идея заключается в нахождении центров кластеров (центроидов) и минимизации суммарного квадратичного отклонения точек кластеров от центров этих кластеров. Центроиды в данном алгоритме выполняют роль центров масс для остальных точек из кластера. Подробное описание алгоритма можно найти [здесь](#).

2. Иерархическая кластеризация  
   Этот метод строит иерархию кластеров. Алгоритм начинает с того, что каждому экземпляру данных сопоставляется свой собственный кластер. Затем два ближайших кластера объединяются в один, и так продолжается, пока не будет образован один общий кластер. Результат иерархической кластеризации может быть представлен с помощью дендрограммы.

   Иерархическая кластеризация менее подходит для работы с большими объемами данных по сравнению с методом k-средних. Это объясняется тем, что временная сложность алгоритма линейна для метода k-средних (O(n)) и квадратична для метода иерархической кластеризации (O(n²)). Кроме того, в кластеризации при помощи метода k-средних алгоритм начинает построение с произвольного выбора начальных точек, поэтому результаты, генерируемые при многократном запуске алгоритма, могут отличаться. В то же время результаты иерархической кластеризации воспроизводимы.

   Метод k-средних хорошо работает, когда форма кластеров является гиперсферической (например, круг в 2D или сфера в 3D), но он более чувствителен к зашумленным данным по сравнению с иерархическим методом.

### Распространение близости (Affinity Propagation)
Распространение близости создает кластеры, отправляя сообщения между парами образцов до схождения. Набор данных описывается с использованием небольшого количества образцов, которые определяются как наиболее репрезентативные для других образцов. Сообщения, отправляемые между парами, представляют пригодность одного образца быть образцом другого, обновляясь в ответ на значения из других пар. Это обновление происходит итеративно до сходимости, после чего выбираются окончательные образцы и, следовательно, дается окончательная кластеризация.

### Средний сдвиг (Mean Shift)
Средний сдвиг — это алгоритм кластеризации, направленный на обнаружение капель в образцах с плавной плотностью. Он работает на основе центроидов, обновляя кандидатов в центроиды, чтобы они были средними точками в данном регионе. Затем эти кандидаты фильтруются на этапе постобработки, чтобы исключить почти дубликаты и сформировать окончательный набор центроидов. Алгоритм автоматически устанавливает количество кластеров, вместо того чтобы полагаться на параметр bandwidth, который определяет размер области для поиска. Этот параметр можно установить вручную или оценить с помощью функции estimate_bandwidth, если полоса пропускания не задана. Алгоритм гарантированно сходится, но прекратит итерацию, когда изменение центроидов станет незначительным.

### DBSCAN
DBSCAN (Density-Based Spatial Clustering of Applications with Noise) — это популярный алгоритм кластеризации, используемый в анализе данных в качестве одной из замен метода k-средних. Метод не требует предварительных предположений о числе кластеров, но нужно настроить два других параметра: eps и min_samples. Эти параметры определяют соответственно максимальное расстояние между соседними точками и минимальное число точек в окрестности (количество соседей), при котором можно говорить о том, что эти экземпляры данных образуют один кластер. В библиотеке scikit-learn есть значения параметров по умолчанию, но их обычно нужно настраивать самостоятельно.

### Теоретическая часть

#### Интерполяция
Интерполяция — это метод нахождения промежуточных значений величины по имеющемуся дискретному набору известных значений. Она использует значения функции, заданные в ряде точек, для предсказания значений между ними. Графически задача интерполирования заключается в построении интерполирующей функции, которая проходит через все узлы интерполирования. Линейная интерполяция является простейшим и часто используемым видом интерполяции, представляя заданные точки прямолинейными отрезками.

#### Аппроксимация
Аппроксимация — это замена одних математических объектов другими, близкими к исходным. В отличие от интерполирования, где интерполирующая функция проходит через узловые точки, аппроксимация позволяет находить дополнительные значения, не проходя через узлы интерполяции.

#### Линейная регрессия
Линейная регрессия — это модель зависимости переменной x от одной или нескольких других переменных с линейной функцией зависимости. Цель линейной регрессии заключается в поиске линии, наилучшим образом соответствующей данным точкам.
